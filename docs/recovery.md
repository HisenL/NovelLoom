# 故障与恢复

- 应用退出：重新执行 `ng serve`，运行状态与 LangGraph checkpoint 均在 SQLite 中。
- 人工门：在“运行与审核”页批准/驳回，或执行 `ng resume <run-id>`。
- 预算暂停：提高项目预算或切换更低费用模型后再恢复；系统不会越过硬预算发起调用。
- Provider 失败：按配置顺序尝试回退链；全部失败后运行标记为 failed，可在修正配置后重试。
- 世界事实编辑：`cascade_recompute` 自动生成事件候选，事件门批准后继续生成多书大纲候选。
- 事件编辑：事件总纲从已批准图谱确定性刷新，并自动生成多书大纲候选。
- 大纲及上游编辑：过期正文仍保留旧版本，任务停在 `rewrite_prose` 确认点，不会后台覆盖。
- 数据备份：应用停止后同时备份主数据库和 checkpoint 数据库。不要只复制其中一个。

当前 `v0.1` 不迁移旧 JSON 图谱或旧原型数据库。
